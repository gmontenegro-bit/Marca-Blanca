App Shell Component

app-shell-component.md

Definir la estructura de navegacion y marco visual del sistema.

Vista rapida

Estructura

  • Topbar (`.topbar`)
  • Sidebar (`.sidebar`)
  • Bottom nav (`.bottom-nav`)
  • Main scroll (`.main-scroll`)

Estados

  • Botones icono hover.
  • Avatar hover con halo.
  • Chip de rol por clase:
  • `.role-chip.admin`
  • `.role-chip.operador`
  • `.role-chip.superadmin`

Variantes

  • Rol operador: menos items de navegacion.
  • Rol admin/superadmin: incluye vistas administrativas.

Criterios

  • Navegacion desktop y mobile no se muestran al mismo tiempo.
  • Estado activo se sincroniza con vista actual.
  • El ancho de sidebar se compensa en `main-scroll` con `margin-left` en desktop.

Estructura

  • Wrapper scroll horizontal: `.tbl-wrap`
  • Tabla base: `table`, `thead`, `tbody`, `th`, `td`
  • Chips de estado para celda de estado

Estados

  • Empty state con bloque `.empty`.
  • Hover en filas para enfoque visual.
  • Filtrado en vivo por input/select (ops y users).

Criterios

  • Encabezados legibles en mobile mediante scroll horizontal.
  • Tipografia monoespaciada para campos de referencia/monto (`.mono`).
  • Empty state visible cuando no hay resultados.

Payment Flow Sheet Component

payment-flow-sheet-component.md

Guiar el pago en pasos cortos dentro de un overlay modal.

Vista rapida

Estructura

  • Overlay: `.overlay` (`#pay-overlay`)
  • Sheet: `.bsheet` (`#pay-sheet`)

Estados

  • Overlay cerrado/abierto: `.open`.
  • Metodo seleccionado: `.method-card.sel`.
  • Factura seleccionada: `.bill-card.sel`.
  • Botones de continuar deshabilitados hasta cumplir requisito de dato.
  • Backdrop click cierra overlay.

Criterios

  • El usuario puede completar el flujo sin salir de contexto.
  • Cada paso muestra titulo y contenido acorde al estado.
  • El comprobante final incluye codigo unico y acciones de enviar/imprimir.

Service Tile Component

service-tile-component.md

Representar servicios pagables en una grilla de seleccion rapida.

Vista rapida

Luz
Agua
Agregar

Estructura

  • Contenedor: `.svc-tile`
  • Icono: `.svc-ico` + `.mi`
  • Nombre: `.svc-name`
  • Subtexto: `.svc-sub`

Estados

  • Default: fondo surface, borde line
  • Hover/active: borde brand soft + elevacion
  • Barra superior decorativa: `::before` con opacidad

Variantes

  • Estandar: servicio existente
  • Agregar: `.add-tile` (borde dashed)

Criterios

  • El grid responde a breakpoints:
  • 2 columnas base
  • 3 columnas >= 480px
  • 4 columnas >= 768px
  • Al hover, el tile no rompe layout y mantiene alineacion vertical.

User Modal Component

user-modal-component.md

Alta y edicion de usuarios desde vista administrativa.

Vista rapida

Estructura

  • Overlay: `#user-overlay`
  • Contenido: `.bsheet`
  • Form fields:
  • nombre
  • apellido
  • tipo de cedula
  • numero de cedula
  • rol
  • contrasena temporal

Estados

  • Modo alta: titulo "Agregar Usuario".
  • Modo edicion: titulo "Editar Usuario" y campos precargados.

Criterios

  • Guardar agrega usuario al listado y muestra snackbar de confirmacion.
  • Editar precompleta los campos del usuario seleccionado.
