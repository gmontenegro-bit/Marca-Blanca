# PayPoint Design Prompts

## Prompt 1 - Mejora de dashboard
"Tomando como base el screen spec de `design/screens/paypoint-screen.md`, proponeme una version mejorada del dashboard (v-dash) manteniendo tokens actuales, mismo sistema de roles y sin frameworks. Inclui variantes mobile/desktop, estados vacios y criterios de aceptacion para handoff FE."

## Prompt 2 - Evolucion del flujo de pago
"Redisenar el flujo `payment-flow-sheet` en 4 pasos maximos, reduciendo friccion en operador. Mantener la identidad visual de PayPoint y los tokens definidos en `design/tokens/paypoint-tokens.md`. Entregar estructura de paso, copy por estado, errores y success state." 

## Prompt 3 - Accesibilidad
"Auditame los componentes `app-shell`, `service-tile`, `data-table` y `user-modal` para mejorar contraste, foco visible, labels y navegacion teclado. Entregame recomendaciones priorizadas y una checklist de acceptance para QA visual + funcional."
